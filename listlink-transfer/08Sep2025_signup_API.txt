# Tighten (high-impact refinements)

1. **API shapes (idempotent + race-safe)**

* `POST /api/lists/:id/items:set` (single endpoint for set/move/replace)
  Body:

  ```json
  { "itemId": "itm_123", "position": 4, "clientRequestId": "uuid-v4" }
  ```

  Returns: `{ slots: [ {position, itemId, title, year, image}, … ] }`
  Notes:

  * Use `clientRequestId` for **idempotency**; dedupe retries.
  * Enforce unique indexes: `(listId, position)` and `(listId, itemId)`.
* `DELETE /api/lists/:id/items?position=4` (removal)
* `POST /api/items/resolve` → returns `{ itemId, slug, title, year, type, image, source, sourceId }` (always normalized).

2. **DB constraints (let the DB help)**

* `UNIQUE(listId, position)` and `UNIQUE(listId, itemId)`; both inside a **transaction**:

  * If `itemId` already on list at `oldPos`, update that row to `position`.
  * If `position` occupied by someone else, delete that row first.
  * Commit only after both conditions hold (prevents ghost dupes under concurrency).

3. **Swap explicitly not supported (yet)**

* You defined **replace/move**—good. Document 409 `CONFLICT_POSITION` if a true swap is attempted by parallel clients; we can add `swap` later without breaking today’s contract.

4. **Auth & runtime**

* You duplicated “Add useSession…” in the checklist—trim it.
* Keep **JWT via `getToken(req)`** for APIs; for UI, wire session once M2 lands.

5. **Caching & search**

* Short-TTL, **q+type** in-memory cache for `/sources/search` (300–600 ms debounce).
* For `/items/resolve`, add a **write-through cache** on `(source, sourceId) → itemId` to avoid thundering herds when many users pick the same popular title simultaneously.

6. **Drafts & autosave (UX resilience)**

* Autosave on any successful slot mutation (server already returns the snapshot).
* On load, **server snapshot wins**; reconcile client state accordingly.

7. **Errors that teach**

* Return small, actionable messages:

  * `DUPLICATE_ITEM`: “Already on your list — moved to slot N.”
  * `CONFLICT_POSITION`: “That spot just changed. Your list is refreshed.”

8. **Observability upgrades**

* Add **request sampling** for `/sources/search` payloads (mask PII), log latency buckets (p50/p95/p99).
* Include `source`, `sourceId`, and `resultCount` in structured logs to debug provider drift.

9. **Test the ugly paths**

* Parallel “add same item to two positions.”
* Rapid re-select spam (proves idempotency).
* External timeout on `/sources/search` (surface graceful skeleton + retry).
